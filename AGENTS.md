# AGENTS.md

このリポジトリで Codex が作業するときの最小ルールです。
目的は「一時退避の戻し漏れ」によるリグレッションを防ぐことです。

## 1. 一時退避の原則

- 原則として、動作経路の一時退避（temporary fallback / bypass）を禁止します。
- 特に「表示品質」「ユーザー操作」「安全ガード」を下げる一時退避は禁止します。

## 2. 例外を使う条件（本当に必要なときだけ）

一時退避を使う場合は、同じ変更内で次を必須にします。

- コード上に `TEMP:` コメントを1か所だけ残す  
  形式: `TEMP: <理由> / 戻す条件> / <期限(YYYY-MM-DD)>`
- この `AGENTS.md` の「Temporary Ledger」に1行追記する
- ユーザー向け報告で「一時退避を入れた」ことを明示する

上記3点のどれか1つでも欠けるなら、その一時退避は不可です。

## 3. マージ前チェック（Definition of Done）

マージ前に必ず次を満たします。

- `TEMP:` が残っていない、または期限内である
- 期限内であっても、次の作業で戻すタスクが明記されている
- 今回の変更で触れた重要経路に「退避前の経路」が復旧している

## 4. Temporary Ledger

運用開始時点: なし
